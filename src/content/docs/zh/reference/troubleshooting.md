## 连接问题

### "您的连接不是私密连接" / SSL 错误
这是最常见的问题，通常意味着 RelayCraft **根证书 (Root CA)** 未正确安装或受信任。

**解决方案:**
1.  从侧边栏打开 **证书**，或前往 **设置 > 证书**。
2.  点击 **安装并信任证书**。
3.  按照系统提示将证书添加到您的 **系统钥匙串** (macOS) 或 **受信任的根证书颁发机构** (Windows)。
4.  **重要提示**: 在 iOS/Android 上，您还必须在设备设置中为已安装的证书启用 "完全信任"。

### 端口 9090 被占用
如果 RelayCraft 无法启动代理，可能是另一个应用程序正在使用端口 9090。

**解决方案:**
1.  前往 **设置** > **网络**。
2.  将 **代理端口** 更改为其他端口 (例如 `8888` 或 `8080`)。
3.  重启 RelayCraft。

### 没有流量出现
如果流量监控列表为空：
1.  检查您的系统或浏览器是否配置为使用代理（RelayCraft **不会自动设置系统代理**，需手动配置）。
2.  确保 **捕获** 已启用（工具栏中的播放/暂停按钮）。
3.  验证您没有过滤掉所有流量（清除过滤栏）。

## 性能问题

### 内存占用过高
如果 RelayCraft 消耗过多内存：
1.  **清除会话**: 点击垃圾桶图标清除当前流量列表。
2.  **禁用重型插件**: 某些可视化插件可能会占用大量资源。
3.  **检查脚本**: 编写不当的 Python 脚本（例如无限循环或在内存中存储大量数据）可能会导致泄漏。

## 规则与脚本

### 规则不工作
-   检查 **优先级**: 规则从上到下执行。较高的规则可能首先匹配并停止执行。
-   检查 **范围**: 确保规则已启用且匹配条件（URL、方法）正确。

### Python 脚本错误
-   检查 **脚本日志**（路径：**设置 > 高级与诊断 > 查看系统日志**）以获取 Python 错误堆栈。
-   确保您的 Python 语法正确。
-   记住在编辑脚本后 **重启代理**。

## 内置日志 (Advanced)

RelayCraft 提供了强大的内置日志查询功能，帮助高级用户诊断问题。

**访问方式**:
1.  前往 **设置** > **高级与诊断**。
2.  点击 **查看系统日志** (View System Logs)。
3.  (可选) 开启 **详细模式** (Verbose Mode) 以记录更底层的核心流量日志。

**日志面板**:
-   **代理日志**: 核心代理服务的运行日志。
-   **系统日志**: 应用程序本身的系统级日志。
-   **脚本日志**: Python 脚本的输出和错误信息。
-   **插件日志**: 已安装插件的运行日志。
-   **崩溃日志**: 应用程序崩溃时的堆栈信息。
